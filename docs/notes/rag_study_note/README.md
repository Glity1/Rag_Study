# 구노(Goono) 전자연구노트 업로드 파일

이 폴더에는 구노 전자연구노트에 업로드하기 위한 연구노트 형식의 문서들이 포함되어 있습니다.

## 파일 구성

1. **01_프로젝트_개요.md** - 프로젝트 전체 개요 및 연구 목적
2. **02_Week1_RAG_기본개념.md** - Week1 연구 활동
3. **03_Week2_PDF_전처리_청킹.md** - Week2 연구 활동
4. **04_Week3_임베딩_벡터인덱스.md** - Week3 연구 활동
5. **05_Week4_RAG_체인구성.md** - Week4 연구 활동 (핵심 모듈)
6. **06_Week5_프롬프트튜닝_LangGraph.md** - Week5 연구 활동
7. **07_Week6_FastAPI_서비스.md** - Week6 연구 활동
8. **08_Week7_Dash_UI.md** - Week7 연구 활동
9. **09_문제해결_기록.md** - 프로젝트 진행 중 문제 해결 기록
10. **10_최종결과_분석.md** - 최종 결과 및 분석

## 사용 방법

### 1. 구노에 업로드
각 마크다운 파일을 구노 연구노트에 복사-붙여넣기 하거나, 내용을 참고하여 직접 작성합니다.

### 2. 첨부 파일 추가
각 주차별 노트에 다음 파일들을 첨부합니다:
- 해당 주차의 소스 코드 파일
- 보고서 및 문서 파일
- 실행 결과 스크린샷
- 실험 데이터

### 3. 전자 서명
각 노트 작성 후 전자 서명으로 완료 표시합니다.

## 업로드 순서 권장

1. 프로젝트 개요 노트 작성
2. Week1~Week7 순서대로 주차별 노트 작성
3. 문제 해결 기록 노트 작성
4. 최종 결과 및 분석 노트 작성

## 주의사항

- 각 노트에 날짜와 시간이 자동으로 기록됩니다
- 중요한 실험 결과는 스크린샷으로 보존하세요
- 코드 파일은 텍스트로 붙여넣거나 파일로 첨부하세요
- 포스트잇 메모를 활용하여 빠른 아이디어를 기록하세요

## 추가 팁

- 화이트보드에 아키텍처 다이어그램을 그려서 사진으로 업로드
- 각 주차별 실행 결과를 포스트잇으로 빠르게 기록
- 문제 해결 과정을 상세히 기록하여 향후 참고

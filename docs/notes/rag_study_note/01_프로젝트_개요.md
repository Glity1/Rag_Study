# RAG Study 프로젝트 개요

## 연구 목적

Retrieval-Augmented Generation(RAG) 워크플로를 7주차 커리큘럼으로 단계화하여 학습하고, 완전한 RAG 파이프라인을 구현하는 것을 목표로 합니다.

## 연구 기간

- 시작일: 2025년 11월 7일일
- 종료일: 2025년 11월 14일

## 연구 목표

1. **PDF 전처리**: 다양한 청킹 전략을 비교 분석하여 최적의 전처리 방법 도출
2. **벡터 검색**: SentenceTransformer 임베딩 및 FAISS 인덱스를 통한 고속 유사도 검색 구현
3. **RAG 체인**: LangChain RetrievalQA 체인을 통한 질의응답 시스템 구축
4. **프롬프트 튜닝**: 다양한 프롬프트 변형 실험을 통한 답변 품질 개선
5. **LangGraph 도입**: LangChain과 LangGraph 비교 분석 및 복잡한 워크플로우 구현
6. **API 서비스**: FastAPI 기반 REST API로 RAG 서비스 제공
7. **웹 UI**: Dash 기반 인터랙티브 질의응답 인터페이스 구축

## 기술 스택

### 핵심 라이브러리
- **LangChain 0.2.16**: RAG 체인 구성
- **LangGraph**: 복잡한 워크플로우 구현
- **SentenceTransformers**: 텍스트 임베딩 생성
- **FAISS**: 벡터 인덱스 및 유사도 검색
- **FastAPI 0.112.1**: REST API 서버
- **Dash**: 웹 UI 프레임워크

### 도구 및 환경
- **Python 3.11**: 개발 언어
- **Hydra**: 설정 관리
- **Docker & Docker Compose**: 컨테이너화 및 배포
- **Gemini API**: LLM 모델 (gemini-2.5-flash)

## 연구 방법론

### 단계별 접근
1. **이론 학습**: RAG 기본 개념 및 관련 기술 학습
2. **단계별 구현**: 주차별로 기능을 단계적으로 구현
3. **실험 및 평가**: 각 단계에서 실험을 수행하고 결과를 평가
4. **문제 해결**: 발생한 문제를 기록하고 해결 방법을 도출
5. **통합 및 배포**: 전체 파이프라인을 통합하고 Docker로 배포

### 실험 설계
- 각 주차별로 명확한 목표 설정
- 실험 결과를 정량적/정성적으로 평가
- 문제 발생 시 상세한 해결 과정 기록

## 주요 성과

1. **완전한 RAG 파이프라인 구현**: PDF부터 UI까지 전체 파이프라인 완성
2. **다양한 청킹 전략 비교**: 5가지 청킹 전략의 장단점 분석

5. **체계적인 문서화**: 아키텍처, 워크플로우, 사용 가이드 문서 작성
